✅ Уже можно вычислять (данные есть)
Производство
Показатель	Источник данных	Комментарий
OEE	AggregatedKPI.oee_estimate	Gateway отдаёт готовый oeeEstimate (0–1), уже сохраняется и показывается в GM Dashboard сравнением с таргетом 75%. Для детального OEE (Availability × Performance × Quality) нужны компоненты — см. ниже.
Процент брака	AggregatedKPI.defect_rate, QualityResult.decision, ProductionOutput.quality_status	Уже вычисляется. GM Dashboard показывает avg_defect_rate, QE Dashboard — defect_pareto, Line Master — defect_rate по сменам.
Время простоев (прокси)	OrderSnapshot (actual_end vs planned_end)	GM Dashboard вычисляет total_delay_hours и delayed_orders — это задержки выполнения заказов. Не является истинным временем простоя оборудования, но близкая прокси-метрика.
Финансы
Показатель	Источник данных	Комментарий
Выручка	AggregatedSales.total_amount, SaleRecord.amount, SalesTrends.total_amount	Полностью доступна. Finance Dashboard отдаёт revenue-trend, sales-breakdown с разбивкой по каналам/регионам/продуктам.
Продажи
Показатель	Источник данных	Комментарий
Объём продаж	AggregatedSales.total_quantity, SaleRecord.quantity	По каналам (channel), регионам, продуктам. Finance Dashboard уже агрегирует.
OTIF	OrderSnapshot (target_quantity, actual_quantity, planned_end, actual_end, status)	Можно вычислить: (заказы со status=completed && actual_quantity >= target_quantity && actual_end <= planned_end) / total_orders * 100. Данных достаточно.

Показатель	Что есть	Чего не хватает
Производительность линии (т/сут)	ProductionOutput.quantity, OrderSnapshot.actual_quantity, смены (shift)	Нет фактического рабочего времени линии (в часах). Можно приблизить через actual_end - actual_start по заказам, но для точного «т/сут» нужны данные о фактических сменах/простоях.
OEE по компонентам	Готовый oeeEstimate от Gateway	Для ручного расчёта OEE = A×P×Q нужны отдельно: Availability (фактическое время / плановое), Performance (фактическая скорость / номинальная скорость линии), Quality (уже есть).

❌ Нельзя вычислить — нужно запрашивать от Gateway (или других систем)
Производство
Показатель	Что нужно от Gateway
Выход масла с тонны семян	Данные о сырье (семена) — входной вес, и о масле как конечном продукте. Сейчас ProductionOutput хранит только выпуск готовой продукции без привязки к сырью. Нужен либо отдельный endpoint сырьевого учёта, либо нормативы выхода.
Время простоев (истинное)	Endpoint с событиями простоя оборудования (downtime events): причина, длительность, линия. Сейчас есть только задержки по заказам.
Финансы
Показатель	Что нужно
Маржинальность	Себестоимость по каждой продаже/продукту. Gateway сейчас отдаёт только amount (выручка), cost отсутствует.
Себестоимость продукции	Затраты (сырьё, энергия, труд, амортизация). Нужен финансовый модуль в Gateway или интеграция с ERP/1С.
EBITDA	Полноценные бухгалтерские данные (операционная прибыль, амортизация, налоги, проценты). Gateway производственный — не содержит таких данных.
Рентабельность по EBITDA	То же самое — нужен EBITDA.
Продажи
Показатель	Что нужно
Доля рынка	Объём рынка по сегментам. Это внешние данные (аналитика рынка, Nielsen, и т.д.), не может прийти из производственного Gateway.
Эффективность промо-акций	Данные о промо-акциях (период, канал, продукт) и затратах на маркетинг. Нужен CRM/маркетинговый модуль в Gateway.

🎯 Рекомендации по приоритетам
1. OTIF — быстрый win. Данных достаточно в order_snapshots, нужен только endpoint вроде /api/v1/orders/otif.
2. Производительность линии — можно сделать прокси-версию через OrderSnapshot (факт/время), но лучше попросить у Gateway данные о сменах или номинальной скорости линий.
3. Финансовые показатели (маржинальность, себестоимость, EBITDA) — требуют отдельного финансового источника. Gateway производственный их не даст. Нужно обсуждать интеграцию с ERP.
4. Выход масла с тонны семян — специфичный производственный показатель. Нужен либо расчётный endpoint в Gateway, либо данные по сырью и нормативам.
