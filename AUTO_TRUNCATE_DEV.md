# Режим разработки: Автоматический TRUNCATE при ошибке

## Назначение

При разработке и исправлении ошибок синхронизации часто нужно:
1. Запустить синхронизацию
2. Если ошибка → очистить БД
3. Исправить ошибку в коде
4. Повторить

Флаг `--auto-truncate` автоматизирует этот процесс.

## Использование

### Полная синхронизация с автоматическим truncate при ошибке
```bash
python run_sync_job.py all --auto-truncate
```

### Что происходит

1. Запускаются все задачи по порядку
2. Если любая задача упадёт с ошибкой:
   - Автоматически запускается `truncate_all_data_task()`
   - Все данные удаляются
   - Цепочка синхронизации останавливается
3. Ты видишь, где именно произошла ошибка
4. Исправляешь код
5. Запускаешь снова: `python run_sync_job.py all --auto-truncate`

## Как это работает

### Без флага (обычный режим)
```bash
python run_sync_job.py all
```
- Запускает все задачи
- Если ошибка → останавливается
- Данные остаются в БД (для отладки)

### С флагом (режим разработки)
```bash
python run_sync_job.py all --auto-truncate
```
- Запускает все задачи
- Если ошибка → автоматический truncate
- Цепочка останавливается
- БД чистая для следующей попытки

## Преимущества

✅ **Быстрая итерация:** Не нужно вручную очищать БД  
✅ **Чистое состояние:** Каждая попытка начинается с пустой БД  
✅ **Видимость ошибок:** Сразу видно, где упала синхронизация  
✅ **Безопасно:** Работает только в режиме разработки  

## Ограничения

- ⚠️ Флаг `--auto-truncate` работает **только** с `all`
- ⚠️ Не работает с отдельными задачами: `python run_sync_job.py sales --auto-truncate` (игнорируется)
- ⚠️ Требует подтверждения для первого truncate

## Альтернативы

### Если нужен truncate без синхронизации
```bash
python run_sync_job.py truncate
```

### Если нужна синхронизация без truncate
```bash
python run_sync_job.py all
```

### Если нужна отдельная задача с truncate
```bash
python run_sync_job.py truncate
python run_sync_job.py sales
```

## Логирование

Все операции логируются:

```bash
tail -f logs/dashboard_api.log | grep -E "(sync_task|truncate)"
```

Output:
```json
{"event": "sync_task_started", "task": "sales", "phase": "entry"}
{"event": "sync_task_failed", "task": "sales", "error": "KeyError: 'totalAmount'"}
{"event": "truncate_all_data_started", "phase": "entry"}
{"event": "truncate_table_done", "table": "aggregated_sales"}
{"event": "truncate_all_data_completed", "phase": "exit", "status": "success"}
```

## Рекомендации

1. **Для разработки:** Используйте `--auto-truncate` для быстрой итерации
2. **Для отладки:** Запускайте без флага, чтобы сохранить данные для анализа
3. **Для production:** Никогда не используйте truncate!
4. **Для тестирования:** Используйте `--auto-truncate` в CI/CD

## Примеры команд

```bash
# Разработка: быстрая итерация
python run_sync_job.py all --auto-truncate

# Отладка: сохранить данные
python run_sync_job.py all

# Одна задача
python run_sync_job.py sales

# Список всех задач
python run_sync_job.py list

# Полный сброс
python run_sync_job.py truncate
python run_sync_job.py all
```
